01 What Are Skills?

Skills are like apps for your AI. You install one and Claude learns a brand new ability.

They work by giving Claude a detailed playbook (a SKILL.md file) with step-by-step instructions. When you need that ability, Claude follows the playbook automatically.

How to install a skill

Drop a folder with a SKILL.md file into one of these locations:

~/.claude/skills/skill-name/// all your projects
.claude/skills/skill-name/// one project only

No downloads. No package managers. Just one file.

How to use a skill

Type /skill-name in Claude Code, or just describe what you want — Claude picks the right skill automatically.

What malicious skills can do
  • Steal your API keys and passwords
  • Read your private files and send them to attackers
  • Run hidden commands on your computer
  • Inject instructions that make Claude do things you didn't ask for
How to stay safe
  1. Stick to official skills first. Anthropic's bundled and official skills are verified safe.
  2. Only install from trusted sources. Check GitHub stars, forks, and recent activity.
  3. Read the SKILL.md file before installing. It's just a text file. Look for anything suspicious.
  4. Never install skills from random links. If someone shares a "cool skill" in Discord or a comment section, verify it yourself first.
  5. Check the allowed-tools field in the SKILL.md. If a skill requests Bash(*) — access to run any command — that's a red flag unless you fully trust the source.

05 Build Your Own Skill

You don't need to be a developer. A skill is just a text file with instructions.

Three steps to create a skill
  1. Create the folder
    mkdir -p ~/.claude/skills/my-skill
  2. Write the SKILL.md file with a name, description, and step-by-step instructions.
  3. Type /my-skill in Claude Code. That's it.
Skill Tips
  • Keep instructions clear and specific. Write them like you're training a new employee.
  • Add disable-model-invocation: true if you only want to trigger it manually.
  • Use $ARGUMENTS to pass in dynamic values.
  • Add supporting files (templates, examples, scripts) in the same folder.
